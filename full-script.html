교사 워크샵 · 강사 풀 스크립트 (80분)

데이터로 시작하는 AI 수업

멘트 한 마디, 클릭 한 번까지. 예상 반응과 대응을 함께 적은 진행 대본.

🎤 강사 멘트(그대로 말하기)🖱️ 행동·화면💭 예상 반응 ↩ 대응
✅ 시작 20분 전 체크리스트 ① 앞 화면(빔)에 greatsong.github.io/map/board.html 전체화면으로. 구글 시트의 지난 데이터행은 미리 비워 0부터 시작.
② 두 번째 브라우저 탭 순서대로 열어 두기: 인사이트 데모 · 인구 실습 교재 · 챗지피티(로그인 상태) · 깃허브(강사 계정 로그인)
③ 조교가 있다면 역할 확인: 실습 때 뒤에서 순회, 막힌 분 옆에 앉기.
④ 마이크·화면공유 점검. 참가자 와이파이 접속 안내문 부착.
1

📍 오프닝, 우리가 지금 데이터를 만든다

⏱ 0–15분

목표: 설명보다 몸이 먼저. "방금 우리가 데이터를 만들고, 지도로 봤다"를 겪게 한 뒤 오늘의 메시지를 선언한다.

0–2분입장 인사, 지도 공개
앞 화면에 전국 지도(board.html)가 떠 있는 상태. QR 코드가 오른쪽 아래에 보인다.
"안녕하세요, 당곡고등학교에서 온 석리송입니다. 오늘 80분, 코딩 수업 아닙니다. 안심하세요. 대신 끝날 때쯤이면 여러분 이름으로 된 진짜 웹사이트가 하나 생겨 있을 겁니다."
"시작하기 전에, 뒤 화면에 지도가 하나 떠 있죠? 지금은 텅 비어 있는데요. 3분 뒤에 이 지도가 어떻게 변하는지 보시죠."
💭 예상 반응 "코딩 안 한다면서 웹사이트를 만든다고?" 하는 반신반의 표정. 웃으며 넘어간다. 지금 설득하지 않는다. 실습이 증명한다.
2–5분QR 스캔, 우리 학교 올리기
"휴대폰을 꺼내 주세요. 화면 오른쪽 아래 QR을 스캔하시면 검색창이 하나 뜹니다. 우리 학교 이름을 검색해서 고르고, 과목과 교육경력을 골라 전송을 눌러 주세요. 딱 30초면 됩니다."
강사도 자기 휴대폰으로 같이 한다(당곡고 검색, 정보·컴퓨터, 경력 선택, 전송). 화면공유는 하지 않고 앞 지도를 계속 비춘다.
💭 "검색해도 우리 학교가 안 나와요" "약칭 말고 정식 명칭으로요. '○○고'가 아니라 '○○고등학교'까지 쳐 보세요. 전국 11,989개 학교가 다 들어 있어서 반드시 나옵니다."
💭 "QR이 안 찍혀요" 주소를 직접 불러 준다. "greatsong.github.io/map 으로 들어오셔도 됩니다." 화면 아래에도 주소가 적혀 있다.
5–8분지도가 살아난다
전송이 들어올 때마다 앞 지도에 핀이 뿅뿅 뜨고, 해당 시군구가 색칠되고, "○○학교 · 과목 · 경력 합류!" 티커가 흐른다. 잠시 말을 멈추고 화면을 보게 둔다.
💭 "어, 우리 학교 떴다!" 웅성웅성, 옆 사람 화면 구경 이 웅성거림이 목적이다. 끊지 말고 10초쯤 그대로 둔다.
"핀이 계속 올라오네요. 왼쪽 아래를 보시면 과목 분포, 경력 분포도 실시간으로 집계되고 있습니다. 오늘은 ○○ 과목 선생님이 제일 많으시네요."
💡 핀이 수도권에 몰리면: "역시 우리 대부분 수도권이네요. 지도가 알아서 확대해 줍니다." 지방 학교가 뜨면 그 학교를 콕 집어 불러 드린다. 환영받는 느낌이 중요하다.
8–11분선언: 방금 우리가 한 일
"방금 여러분이 스마트폰으로 넣은 정보가, 실시간으로 앞 화면 지도에 그려졌죠? 방금 우리는 데이터를 만들고, 모으고, 시각화했습니다. 오늘 80분 동안 할 일이 정확히 이겁니다. 이번엔 여러분 손으로요."
"AI 시대라고 하죠. 그런데 AI는 열매를 다루는 기술이에요. 번역, 요약, 그림, 코드. 전부 화려한 열매입니다. 그 열매를 맺게 하는 뿌리는 뭘까요? 데이터입니다. 여전히, 데이터예요."
💭 고개 끄덕임, 몇 분은 메모 시작 "열매와 뿌리"는 오늘 세 번 반복할 문장이다. 여기서 또박또박 한 번.
11–15분오늘의 여정, 준비물 점검
"오늘은 코딩을 배우는 시간이 아니라 데이터로 수업을 상상하는 힘을 깨우는 시간입니다. 이건 마중물이에요. 오늘 감을 잡고, 학교로 돌아가서는 여러분 수업의 데이터로 이어 가시는 겁니다."
허브(greatsong.github.io/workshop/)를 잠깐 띄워 오늘 순서를 보여 준다: 질문으로 시작 → 앱 만들기 실습 → 학교로 가져갈 미션.
"준비물 확인 하나만 할게요. 깃허브 계정 만들어 오신 분, 손 들어 주세요."
💭 절반쯤 손을 든다. 나머지는 눈치를 본다 "아직인 분들, 전혀 문제없습니다. github.com 들어가서 Sign up, 메일 인증까지 딱 5분이면 돼요. 다음 순서 듣는 동안 슬쩍 만들어 두시면 실습 때 같이 완성하실 수 있습니다. 정 안 되면 오늘은 편하게 구경만 하셔도 충분히 가져가실 게 있어요."
2

🔎 질문으로 시작, 데이터가 답한다

⏱ 15–35분

목표: "감이 아니라 데이터로 답한다"의 짜릿함. 데이터 상상력에 불을 붙인다.

15–17분질문 하나 던지기
"질문 하나 드릴게요. 우리 학교 동네엔 몇 살이 많을까요? 어르신이 많은 동네일까요, 아이들이 많은 동네일까요? 그리고… 청소년이 유독 많은 동네가 있다면, 그게 혹시 학군지일까요? 감으로 말고, 데이터로 봅시다."
인사이트 데모(greatsong.github.io/map/insight.html)로 화면 전환.
💭 "학군지" 단어에 피식 웃음 학군지는 모든 교사가 아는 단어다. 이 웃음이 몰입의 시작.
17–21분시연 1: 대치1동의 피라미드
검색창에 대치1동 입력. 인구 피라미드와 지표 카드가 뜬다. 잠깐 말을 멈추고 모양을 보게 한다.
"서울 강남구 대치1동입니다. 피라미드 모양이 특이하죠? 가운데, 그러니까 10대 구간이 불룩 튀어나와 있어요. 지표를 보시면… 만 14세 비율이 전국 2,762개 동 중에 1위입니다. 중2가 전국에서 제일 많은 동네예요. 왜 그럴까요?"
💭 "대치동이니까…" "학원 때문이죠" 즉답이 나온다 "맞습니다. 우리는 답을 알고 있었죠. 그런데 지금은 데이터가 그걸 그려서 보여 준다는 게 다릅니다."
21–24분시연 2: 쌍둥이 동네 (탄성 지점)
"그럼 이런 질문은 어때요? 전국에서 대치1동이랑 인구 구조가 가장 닮은 쌍둥이 동네는 어디일까요?"
화면을 아래로 스크롤해 '쌍둥이 동네' 목록을 보여 준다. 하나씩 짚으며 읽는다.
"평촌 귀인동, 노원 중계1동, 대전 둔산1동, 대구 수성구… 어떠세요? 전부 학군지입니다. 저는 이 동네들 이름을 넣은 적이 없어요. 인구 구조 숫자만 비교했을 뿐인데, 데이터가 학군지를 알아봅니다."
💭 "우와" 하는 탄성, 사진 찍는 분들 오늘의 최고 탄성 지점. 3초 쉬고, 사진 찍을 시간을 준다.
24–27분시연 3: 해석은 AI에게
"모양은 봤는데, 이걸 말로 풀어 주는 건 AI가 잘합니다. 여기 '이 동네 분석 프롬프트 복사하기' 버튼이 있어요."
버튼 클릭 → 챗지피티 탭으로 전환 → 붙여넣고 전송 → 나온 해석을 소리 내어 읽는다(두세 문장만).
"이 흐름을 기억해 주세요. 데이터가 그리고, AI가 해석하고, 판단은 사람이 한다. 오늘 실습도 정확히 이 순서로 갑니다."
💭 "저 버튼 나도 눌러 보고 싶다"는 눈빛 "지금부터 직접 하실 시간입니다"로 바로 연결.
27–33분참가자 활동: 우리 동네 넣어 보기
"이제 여러분 차례입니다. 같은 주소로 들어가서 우리 동네, 혹은 우리 학교 동네를 검색해 보세요. 그리고 발견한 것 한 가지를 옆 선생님과 나눠 주세요. 5분 드립니다."
앞 화면에 데모 주소를 크게 띄워 둔다. 강사와 조교는 돌아다니며 화면을 들여다본다.
💭 "우리 동네를 검색했는데 안 나와요" "행정동 이름으로 검색됩니다. 아파트 단지나 역 이름 말고 '○○1동'처럼요. 동 이름이 헷갈리면 구 이름 먼저 쳐 보세요."
💭 "우리 동네 고령화율이 이렇게 높아요?" 곳곳에서 놀람 "그 발견 잠시 뒤에 전체 공유해 주세요!" 하고 예약을 걸어 둔다.
33–35분공유 두세 개 받고 실습 예고
"발견한 것, 두 분만 들어 볼게요."
손 든 사람 2명(없으면 순회 때 봐 둔 사람을 지목). 각 발견에 한 문장씩 반응한다.
"좋습니다. 그런데 지금 쓰신 이 데모, 누가 만들었을까요? 저는 코드를 한 줄도 안 썼습니다. 말로 시켜서 만들었어요. 지금부터 35분, 여러분이 직접 만드십니다."
💭 "내가 이걸 만든다고?" 하는 술렁임 술렁임을 즐기며 교재 화면으로 전환.
3

🛠 실습, 우리 동네 인구 분석 앱 만들기

⏱ 35–70분

목표: 말로 설명 → AI가 코드 → 깃허브 배포. 내 손으로 데이터 앱을 만든다. 낙오 방지선은 교재의 "완성 코드" 탭.

35–37분교재 열기, 규칙 안내
앞 화면에 실습 교재(greatsong.github.io/ingu/) 표지. 참가자도 각자 접속하게 한다.
"교재 주소는 화면과 같습니다. 게임처럼 레벨 1 배포, 레벨 2 데이터, 레벨 3 시각화, 레벨 4 대시보드 순서로 올라갑니다. 규칙은 딱 두 개예요. 첫째, 코드를 이해하려고 하지 마세요. 오늘은 복사와 붙여넣기만 있으면 됩니다. 둘째, 막히면 각 레벨의 '완성 코드' 탭을 여세요. 그대로 붙여넣으면 무조건 됩니다."
💭 "코드를 이해하지 말라고요?" 웃음 "네. 이해는 AI가 합니다. 우리는 주문하는 쪽이에요." 바이브코딩 한 줄 정의를 여기서 심는다.
37–47분LV1 배포: 내 주소 만들기 (화면공유로 다 함께)
여기서부터 강사 화면공유. 한 클릭마다 소리 내어 말하며 천천히 따라오게 한다.
① github.com 로그인 → 오른쪽 위 + → New repository
② 이름 population · Public · Add a README file 체크 → Create repository
③ Add file → Create new file → 파일 이름 index.html
④ 교재 레벨 1의 코드(복사 버튼) 붙여넣기 → 초록 Commit changes
⑤ Settings → 왼쪽 Pages → Branch를 main으로 → Save
⑥ 1~2분 뒤 뜨는 내 주소(내아이디.github.io/population)를 새 탭에서 열기
"깃허브의 모든 '만들기' 버튼은 초록색입니다. 뭘 눌러야 할지 모르겠으면 초록 버튼을 찾으세요."
"주소가 바로 안 열려도 정상입니다. 첫 배포는 1~2분 걸려요. 404가 떠도 당황하지 말고 잠시 뒤 새로고침."
💭 "Pages 메뉴가 안 보여요" "Settings 화면에서 왼쪽 목록을 아래로 내려 보세요. Code and automation 묶음 안에 Pages가 있습니다."
💭 "404 Not Found가 떠요" "배포가 아직 안 끝난 거예요. 1분 기다렸다 새로고침. 그래도 안 되면 Pages에서 Branch가 main으로 저장됐는지 다시 확인."
💭 "README 체크를 빼먹었어요" 괜찮다. Add file → Create new file로 index.html을 만들면 그대로 진행된다.
45분쯤: "화면에 '🏘️ 우리 동네 인구 구조' 글자가 내 주소로 보이는 분, 손!" 과반이 들면 진행. 못 든 분 옆에 조교를 붙인다.
"축하합니다. 방금 여러분은 전 세계 누구나 접속할 수 있는 웹사이트의 주인이 됐습니다. 오늘 배포는 이걸로 끝이에요. 이제부터는 이 파일 하나만 고쳐 나갑니다."
47–55분LV2 데이터: AI에게 첫 주문
교재 레벨 2. 먼저 '🤖 AI로 직접' 탭 기준으로 시연한다.
① 인구 데이터 주소 복사 → 새 탭에서 열어 진짜 CSV임을 보여 준다("3,900개 동네가 다 들어 있습니다")
② AI 프롬프트 복사 → 챗지피티에 붙여넣고 맨 끝에 데이터 주소 추가 → 전송
③ AI가 준 코드 전체 복사 → 깃허브 index.html 연필(✏️) → 기존 내용 다 지우고 붙여넣기 → Commit
④ 내 주소 새로고침 → 검색창에 동네 입력 → 총인구·남녀 인구 등장
"프롬프트를 읽어 보세요. 코드가 한 줄도 없죠? 원하는 걸 한국어로 설명했을 뿐입니다. 이게 바이브코딩이에요."
💭 "제 화면은 강사님 것과 다르게 생겼어요" "정상입니다! AI는 같은 주문에도 매번 조금씩 다르게 만들어요. 식당 주방장이 바뀐 거라고 생각하세요. 기능만 되면 성공입니다." 이 말이 이 블록에서 제일 중요하다.
💭 "붙여넣었는데 화면이 하얗게 나와요" 십중팔구 코드 일부만 복사했거나, 기존 내용을 안 지우고 이어 붙인 경우. "완성 코드 탭을 열어 통째로 다시 붙여넣으세요"로 즉시 복구.
💭 "AI가 코드를 여러 조각으로 줬어요" "'전체 코드를 한 파일로 다시 줘'라고 한 줄 보내세요. AI에게 다시 시키는 것도 실력입니다."
55–62분LV3 시각화: 인구 피라미드
교재 레벨 3. 이어서 대화 프롬프트 복사 → 아까 그 챗지피티 창에 이어서 전송 → 코드 교체 → Commit → 새로고침. 피라미드가 뜬 순간 화면을 크게.
"아까 데모에서 봤던 그 피라미드가, 지금 여러분 주소에서 그려지고 있습니다. 대치1동도 넣어 보세요. 아까 그 불룩한 10대, 이제 내 앱에서 보입니다."
💭 여기저기서 "떴다!" 소리, 옆자리에 화면 자랑 실습의 절정. 잘 된 화면 두어 개를 앞에서 크게 비춰 준다(본인 허락 받고).
💭 "그래프 자리가 비어 있어요" "새로고침을 Ctrl+Shift+R로 세게 한 번. 그래도 안 되면 완성 코드 탭." 원인 규명은 하지 않는다. 시간이 금이다.
62–67분LV4 대시보드: 마지막 단장
교재 레벨 4. 밝은 대시보드 프롬프트 이어 보내기 → 코드 교체 → Commit → 새로고침. 지표 카드 4개와 테마 버튼(크림·민트·베리)이 생긴다.
"이제 뉴스에 나올 법한 대시보드가 됐죠? 여유가 되는 분은 딱 하나만 더 시켜 보세요. '우리 학교 상징색으로 테마를 하나 추가해줘'. 말하는 대로 바뀝니다. 단, 하나만요. 욕심은 마무리 뒤에!"
💭 진도 차이가 크게 벌어져 있다 (LV2인 분, LV4 끝낸 분) "지금 어느 레벨이든 괜찮습니다. LV1까지만 됐어도 오늘 최고 수확인 '내 주소'는 이미 손에 있어요." 빠른 분에게는 추가 주문 도전을, 느린 분에게는 완성 코드 탭을.
67–70분실습 마감, 손들기
"자, 손 한번 들어 볼까요? 피라미드까지 뜬 분? … 대시보드까지 간 분? … 대단하십니다. 못 끝낸 분들, 교재는 계속 열려 있으니 오늘 밤 완성 코드로 5분이면 따라잡으실 수 있어요."

70–73분다음 미션: 우리 학교 급식 투표 앱
"오늘은 공공데이터로 감을 잡았어요. 이제 진짜는, 학교로 돌아가 우리 학교, 우리 반의 데이터로 해보는 겁니다. 성적, 출결, 설문, 활동 기록, 아이들의 관심사까지."
"그래서 다음 미션을 드립니다. 학교에서 우리 학교 급식 투표 앱을 만들어 보세요. 나이스에 올라오는 진짜 급식 데이터를 불러와서, 학생들이 좋아하는 메뉴에 투표하는 앱입니다. 오늘과 똑같은 방식, 주소는 greatsong.github.io/meal. 오늘의 마중물이 진짜 물길이 되는 순간이에요."
급식 교재 표지를 10초만 보여 준다. 온라인 협동판(meal-db)과 시트 연동 가이드(gas)는 링크만 언급.
